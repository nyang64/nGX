# Auth Service

**Module**: `nGX/services/auth`
**Port**: `8081`
**Role**: API key lifecycle management and validation. The single source of truth for all authentication.

## API Key Design

### Format

Keys are generated by `auth.GenerateAPIKey()` which produces:

```
am_live_<base64url(32 random bytes)>
```

Example: `am_live_abc123XYZdef456GHIjkl789MNOpqr012STU=`

A short display prefix (e.g. `am_live_ab12`) is stored alongside the hash for identification in the UI without exposing the key.

### Storage

Only the **SHA-256 hex hash** of the plaintext key is stored in the database (`key_hash` column). The plaintext is returned **once** at creation time and never stored. This means:

- Key lookup is O(1) — hash the input, query by hash with a B-tree index.
- A database breach does not expose usable key material.
- SHA-256 is used (not bcrypt) because key validation happens on every API request and needs to be fast. The 32-byte random key provides sufficient entropy that offline brute-force is not feasible regardless of the hash algorithm.

### Key Scoping

A key can be:

- **Org-wide** (`pod_id = NULL`): can access all inboxes in the org.
- **Pod-scoped** (`pod_id = <uuid>`): restricted to a single pod. The API Gateway forwards `X-Pod-ID` to internal services, which use `Claims.CanAccessPod()` to enforce the restriction.

## Endpoints

### POST /api-keys

Creates a new API key. Returns the plaintext key in the `key` field — this is the only time it is visible.

**Request**:
```json
{
  "org_id": "uuid",
  "name": "Production Agent",
  "scopes": ["inbox:read", "inbox:write"],
  "pod_id": null
}
```

At least one scope is required; the service returns an error if `scopes` is empty.

**Response** (201):
```json
{
  "id": "uuid",
  "name": "Production Agent",
  "key": "am_live_...",
  "key_prefix": "am_live_ab12",
  "scopes": ["inbox:read", "inbox:write"],
  "created_at": "..."
}
```

### GET /api-keys?org_id={id}

Lists all active (non-revoked) keys for an org. Never returns the `key` value — only metadata: `id`, `name`, `key_prefix`, `scopes`, `pod_id`, `last_used_at`, `created_at`.

### GET /api-keys/{keyID}?org_id={id}

Returns a single key by ID, scoped to the org.

### DELETE /api-keys/{keyID}?org_id={id}

Revokes a key by setting `revoked_at = NOW()`. The operation is a soft delete — the row stays in the table but `Validate` will no longer return it. Revocation is immediately effective; all subsequent validation calls return 401.

### POST /validate (internal)

Called by the API Gateway on every authenticated request.

**Request**: `{"key": "am_live_..."}`
**Response** (200): Claims JSON
**Response** (401): `{"error": "invalid key"}`

**Implementation** (`PostgresAPIKeyStore.Validate`):

1. Computes `SHA-256(plaintextKey)` via `auth.HashAPIKey`.
2. Queries:
   ```sql
   SELECT ... FROM api_keys
   WHERE key_hash = $1
     AND revoked_at IS NULL
     AND (expires_at IS NULL OR expires_at > NOW())
   ```
3. Updates `last_used_at = NOW()` asynchronously (non-fatal if it fails).
4. Returns `Claims{OrgID, KeyID, Scopes, PodID}`.

## RLS Behavior

The `Validate` endpoint uses a **direct pool connection** — it bypasses RLS. This is intentional and safe: the org is unknown before the key is looked up, so we cannot set `app.current_org_id` first. The query only reads by the globally-unique `key_hash` column.

All other endpoints (`Create`, `List`, `Revoke`, `GetByID`) use `db.WithOrgTx()` which sets the RLS context variable.

## RBAC Scopes

Scopes are defined in `nGX/pkg/auth/scopes.go`.

| Scope | Grants |
|-------|--------|
| `org:admin` | Wildcard — `Claims.HasScope()` returns true for any scope check |
| `pod:admin` | Create and manage pods |
| `inbox:read` | Read inboxes, threads, messages |
| `inbox:write` | Send messages, create inboxes, update threads |
| `draft:read` | View drafts |
| `draft:write` | Create/update/approve/reject drafts |
| `webhook:read` | View webhooks and delivery history |
| `webhook:write` | Create/update/delete webhooks |
| `search:read` | Full-text search |

`org:admin` is a wildcard: a key with only `org:admin` in its scopes passes every `requireScope` check.
